# 穿搭预览应用 PRD (Product Requirements Document)

## 1. 项目概述

### 1.1 项目背景
用户在购买衣物时经常面临搭配困难，无法直观地看到衣物在自己身上的实际效果。传统的购物方式依赖于用户的想象力和经验，导致购买决策困难，退货率高。

### 1.2 项目目标
开发一个基于AI虚拟试衣技术的Web应用，允许用户上传个人照片和衣物图片，通过阿里云DashScope AI生成真实的试衣效果，帮助用户做出更明智的购买决策。

### 1.3 价值主张
- **真实效果预览**：AI生成的试衣效果更接近真实上身效果
- **减少退货率**：预览效果后再购买，降低因不合适导致的退货
- **节省时间**：无需反复试穿，快速查看多种搭配效果
- **双模式选择**：支持替换模式和叠加模式，满足不同场景需求

## 2. 用户画像

### 2.1 主要用户群体
- **时尚爱好者**：喜欢尝试不同风格，经常购买新衣物的年轻人
- **网购达人**：频繁在线购物，对搭配有一定需求
- **忙碌白领**：时间有限，希望快速找到合适的搭配
- **时尚博主**：需要预览穿搭效果进行内容创作

### 2.2 用户场景
- **购物前预览**：在电商平台看到心仪衣物，上传到应用查看效果
- **搭配规划**：尝试不同衣服的搭配效果
- **层次穿搭**：使用叠加模式测试多件衣服的层次效果
- **社交分享**：将试衣效果分享到社交媒体

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 用户管理
- **用户注册/登录**
  - 支持邮箱注册/登录
  - JWT token认证
  - 个人信息管理（头像、邮箱等）

#### 3.1.2 AI虚拟试衣
- **步骤化流程**
  - 步骤1：上传人物照片（上半身，清晰照片效果更好）
  - 步骤2：上传衣服照片（平铺或衣架拍摄）
  - 步骤3：选择试衣模式
  - 步骤4：AI生成试衣结果

- **试衣模式**
  - **替换模式**：直接替换身上衣服
  - **叠加模式**：保留原衣服，新衣穿在外面（需额外上传要保留的衣服）

- **衣服类型**
  - 上装（上衣、外套、T恤等）
  - 下装（裤子、裙子等）

#### 3.1.3 试衣历史管理
- **历史记录**
  - 自动保存每次试衣结果
  - 显示人物照、衣服照、结果图
  - 显示试衣模式和衣服类型
  - 显示创建时间

- **操作功能**
  - 下载试衣结果图片
  - 删除历史记录

#### 3.1.4 首页预览
- **快速预览**：上传人物照和衣服照快速查看效果
- **简单流程**：简化版试衣流程

### 3.2 辅助功能

#### 3.2.1 图片处理
- **自动压缩**：自动压缩图片至合适大小
- **格式支持**：JPG、PNG格式
- **大小限制**：最大10MB

#### 3.2.2 用户体验优化
- **进度指示**：圆形进度条显示AI处理进度
- **加载状态**：图片上传和处理时显示加载动画
- **错误处理**：提供重试和重新开始选项
- **响应式设计**：移动端和桌面端适配

## 4. 技术需求

### 4.1 前端技术栈
- **框架选择**：Next.js 16.1.6 (App Router)
- **UI框架**：React 18
- **状态管理**：React Hooks (useState, useEffect, useCallback)
- **样式方案**：Tailwind CSS
- **类型安全**：TypeScript

### 4.2 后端技术栈
- **主框架**：Next.js API Routes
- **数据库**：Prisma ORM + SQLite
- **认证方式**：JWT (httpOnly cookies)
- **AI服务**：阿里云 DashScope AI试衣 API

### 4.3 数据模型

#### 用户表（User）
```typescript
- id: 主键 (cuid)
- email: 邮箱
- name: 姓名
- password: 密码哈希
- avatar: 头像
- height: 身高
- weight: 体重
- bodyType: 体型
- createdAt: 创建时间
- updatedAt: 更新时间
- 关联: clothingItems, outfits, preferences, tryOnHistory
```

#### AI试衣历史表（TryOnHistory）
```typescript
- id: 主键 (cuid)
- userId: 用户ID
- personImageUrl: 用户照片URL
- clothImageUrl: 衣服照片URL
- keepClothImageUrl: 保留的衣服照片URL（叠加模式）
- resultImageUrl: AI生成结果URL
- clothType: 衣服类型 (upper, lower)
- tryOnMode: 试衣模式 (replace, overlay)
- createdAt: 创建时间
```

## 9. 风险评估

### 9.1 技术风险
- **AI API稳定性**：依赖第三方AI服务，需做好错误处理
- **图片处理精度**：AI生成效果可能不理想
- **API成本**：AI调用可能产生费用

### 9.2 商业风险
- **用户获取成本**：获客成本可能较高
- **竞争压力**：面临其他试衣应用的竞争
- **AI费用**：需要找到合适的定价模式

### 9.3 风险应对
- **错误处理**：完善的错误提示和重试机制
- **用户反馈**：持续收集反馈优化效果
- **成本控制**：监控API使用量
