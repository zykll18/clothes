# 后端整改 Roadmap

## 1. 当前状态概览

当前项目后端已经具备可运行的 MVP 主链路：

- 用户注册、登录、登出、登录态查询
- 衣橱新增、列表、删除
- AI 试衣任务创建与状态轮询
- 试衣历史保存、列表、删除

但它仍然更接近“可演示原型”，还没有达到“后端边界稳定、可持续迭代”的状态。当前主要问题集中在四类：

- 安全边界不一致
- 路由层认证与校验模式不统一
- 数据与业务语义仍有漂移风险
- 上线所需的可观测性、存储和测试能力不足

## 2. P0：安全与接口一致性

### 2.1 目标

在不扩大功能范围的前提下，把当前最容易出问题的后端边界先收紧，确保“能用”至少是“相对安全、相对一致地能用”。

### 2.2 范围

- 给 `/api/ai-tryon` 和 `/api/ai-tryon/status` 增加服务端登录校验
- 给高风险接口增加基础防滥用控制
- 把删除类接口统一到显式 `verifyToken` 类型收口，不再保留本地 `JWTPayload` cast
- 明确当前 P0 口径下的试衣历史保存语义：
  - `tryOnMode` 当前主流程固定为 `replace`
  - `clothType` 当前主流程按页面现有模式落在 `upper / lower`

### 2.3 涉及文件

- `app/src/app/api/ai-tryon/route.ts`
- `app/src/app/api/ai-tryon/status/route.ts`
- `app/src/app/api/auth/login/route.ts`
- `app/src/app/api/auth/register/route.ts`
- `app/src/app/api/clothing/[id]/route.ts`
- `app/src/app/api/tryon-history/[id]/route.ts`
- `app/src/lib/jwt.ts`
- 可能新增共享辅助文件，例如认证或限流工具

### 2.4 风险

- 若限流做得太激进，会影响本地调试和正常试衣
- 若 AI 状态轮询限流策略不合理，会导致前端误判为后端异常
- 若直接在每个路由各写一套保护逻辑，后续维护成本仍然偏高

### 2.5 验收标准

- 未登录调用 `/api/ai-tryon` 和 `/api/ai-tryon/status` 返回 `401`
- 登录用户仍可完成现有试衣闭环
- 注册、登录、AI 试衣接口存在基础限流保护
- `clothing/[id]` 和 `tryon-history/[id]` 不再使用本地 `JWTPayload` cast
- 全量 `lint` 和 `tsc --noEmit` 通过

## 3. P1：资料、存储与可观测性

### 3.1 目标

把当前“能跑”的数据层进一步整理成“可维护”的数据层，补上用户资料编辑、图片存储边界和任务可追踪性。

### 3.2 范围

- 增加用户资料更新接口
- 为图片存储方案定正式口径，不再长期停留在“字符串/URL 混合体”
- 为 AI 试衣任务增加后端可追踪能力
- 统一列表接口的分页与筛选边界

### 3.3 涉及主题

- `/api/auth/me` 的扩展或新增资料接口
- 衣橱与历史记录中的图片字段
- AI 任务生命周期记录
- 分页上限与查询参数验证

### 3.4 风险

- 一旦引入正式图片存储，会牵动前后端多个路径
- 任务持久化若设计过重，容易把原型推向过度工程化

### 3.5 验收标准

- 用户可以更新基础资料
- 图片来源和存储路径在文档与代码里一致
- AI 任务失败可以被后端追踪与排查
- 历史和衣橱查询边界明确且有上限

## 4. P2：未落地模型与上线准备

### 4.1 目标

处理当前 schema 中“已经建模但未落地”的部分，并为未来部署和长期维护做准备。

### 4.2 范围

- 决定 `Outfit / OutfitItem / UserPreference` 是继续做还是从 schema 中移除
- 增加后端测试基线
- 规划 SQLite 之后的数据库演进路径
- 补充更系统的监控、日志和错误分级

### 4.3 风险

- 这部分最容易范围膨胀
- 如果不先完成 P0/P1，P2 的很多准备工作会建立在不稳定边界上

### 4.4 验收标准

- 未落地模型有明确去留决策
- 核心 API 至少具备基础测试覆盖
- 数据库演进路线在文档中明确

## 5. 推荐执行顺序

1. 先完成 P0，把安全边界和接口一致性收紧
2. 再进入 P1，补资料、存储和任务可追踪性
3. 最后做 P2，决定未落地模型并补上线准备

## 6. 本轮实现边界

本轮只进入 P0，不实现 P1/P2。

本轮明确不做：

- 用户资料编辑
- 图片持久化方案升级
- AI 任务数据库持久化
- Outfit / Preference 相关 API
- 后端测试体系搭建
