# 潮流穿搭博主内容预演工具重定位设计

## 1. 背景

当前 `clothes` 项目原始定位偏“通用 AI 试衣 / 购物决策工具”，但新的目标用户已经收窄为潮流穿搭博主，产品核心价值也从“试衣判断是否合身”转向“发布内容前的风格预演与方向筛选”。

这次设计的目标不是重做整个平台，而是在现有项目基础上完成一次明确的产品重定位，并为后续实现提供可执行边界。

## 2. 新定位

### 2.1 一句话定义

这是一个帮助潮流穿搭博主在发抖音 / 小红书前，先预演同一套 `look` 不同内容方向的 AI 工具。

### 2.2 第一版用户

- 抖音 / 小红书潮流穿搭博主
- 偏街头、老钱、clean fit、美式复古等潮流属性
- 需要更高频地产出日常内容方向

### 2.3 第一版价值

- 帮助用户决定这次内容该发哪一版
- 帮助用户在拍摄前筛掉不出片的方向
- 沉淀个人风格内容案例，用于后续拍摄或品牌沟通

## 3. 推荐主流程

第一版采用 `Hybrid` 交互方案，主流程如下：

1. 上传本人照片
2. 选择今天想穿的主色
3. 系统根据主色混排推荐用户素材和系统预置单品
4. 进入按部位选衣服阶段
5. 以滚动卡片方式逐层选择外套、内搭、裤子、项链 / 配饰、鞋子
6. 完成 `look` 后发起内容预演
7. 返回 3 个方向结果并进行比较
8. 选定主推版本，保存并下载

## 4. 页面结构

### Step 1：上传人物

- 上传本人照片
- 提示用户使用正面、清晰、轮廓明确的人像

### Step 2：选择主色

- 页面展示“今天想搭配的颜色”
- 使用大调色盘让用户选择今天想穿的主色
- 这一步是轻决策，目的是缩小后续候选范围

### Step 3：推荐衣服

- 根据主色混排推荐两类素材：
  - 用户自己上传过的衣服素材
  - 系统预置潮流单品
- 这里的目标是建立候选池，而不是直接完成搭配

### Step 4：按部位滚动卡片选衣服

- 部位顺序：
  - 外套
  - 内搭
  - 裤子
  - 项链 / 配饰
  - 鞋子
- 每个部位都使用中心聚焦的滚动卡片
- 中间卡片最大、最亮，两侧卡片退后缩小，形成 2.5D 仪式感

### Step 5：内容预演结果

- 输出 3 个方向结果
- 每张图带方向标签
- 用户选定本次主推版本
- 支持保存、下载、回看历史

## 5. 功能边界

### 5.1 第一阶段必须做

- 上传本人照片
- 选择今天主色
- 按主色混排推荐用户素材和系统单品
- 按部位滚动卡片选衣服
- 生成 3 个内容方向结果
- 选定主推版本
- 保存、下载、回看历史

### 5.2 第一阶段明确不做

- 真 3D 建模或复杂骨骼交互
- 精准尺码和版型判断
- 品牌协作后台
- 自动文案、自动选题、自动发帖
- 复杂社交系统
- 大规模推荐算法

## 6. 对现有代码的实现建议

### 6.1 优先复用

- 登录 / 注册
- 当前 `/tryon` 页面骨架
- 历史记录页
- 下载与删除能力
- 现有偏 editorial 的视觉语言

### 6.2 首批改造重点

- 首页定位与文案
- `/tryon` 流程，从“上传衣服直接生成”改成“主色 -> 推荐 -> 按部位滚动选择 -> 多结果比较”
- 结果页，从单结果展示改成 3 图比较
- 历史页，从试衣历史改成内容预演历史

## 7. 数据语义调整

第一版只补必要结构，不做过重建模：

- `ColorSelection`
  - 记录本次内容预演选择的主色

- `PreviewSession`
  - 代表一次完整的内容预演流程
  - 记录主色、所选各部位单品、最终主推版本

- `PreviewVariant`
  - 代表同一次 session 下的 3 个方向结果
  - 记录方向标签、结果图、排序、是否被选中

- `WardrobeItem`
  - 现有衣物表先尽量复用
  - 前端语义改成用户素材库

## 8. 风险与控制

主要风险：

- 步骤过长导致用户疲劳
- 颜色选择和推荐结果不匹配
- 滚动卡片如果质感不足，会退化成普通轮播

控制策略：

- 第一版先做 2.5D，而不是真 3D
- 把颜色选择和滚动卡片分成不同阶段
- 先让流程顺畅，再逐步升级动效和视觉层次

## 9. Agent 执行口径

后续实现 agent 必须统一按以下口径工作：

- 这是一个潮流穿搭博主内容预演工具
- 第一版目标用户是潮流穿搭博主，不是普通消费者
- 第一版起点是“今天想穿的颜色”
- 第一版流程是“颜色 -> 推荐 -> 按部位滚动选衣服 -> 内容预演”
- 第一版重点是帮助用户决定这次内容该发哪一版
